NewMarkdownViewer-module__safe-html-box__ZT1eD"><h2 dir="auto">Problem</h2>
<p dir="auto">The current parser only recognizes a limited subset of ANTLR v4 syntax. When parsing grammars like XMLLexer.g4, the parser fails to recognize rules that use modern ANTLR constructs, resulting in an empty <code class="notranslate">rules</code> array in the AST.</p>
<h2 dir="auto">Current Limitations</h2>
<p dir="auto">The <code class="notranslate">rule_element</code> definition currently only supports:</p>
<ul dir="auto">
<li>String literals (<code class="notranslate">'...'</code> or <code class="notranslate">"..."</code>)</li>
<li>Character classes (<code class="notranslate">[a-zA-Z]</code>)</li>
<li>Identifiers (rule names)</li>
</ul>
<h2 dir="auto">Missing Features</h2>
<h3 dir="auto">1. Quantifiers</h3>
<ul dir="auto">
<li><code class="notranslate">*</code> (zero or more)</li>
<li><code class="notranslate">+</code> (one or more)</li>
<li><code class="notranslate">?</code> (optional)</li>
<li><code class="notranslate">{n,m}</code> (bounded repetition)</li>
<li><code class="notranslate">*?</code> (lazy zero or more)</li>
<li><code class="notranslate">+?</code> (lazy one or more)</li>
</ul>
<h3 dir="auto">2. Special Sequences</h3>
<ul dir="auto">
<li><code class="notranslate">.</code> (any character)</li>
<li>Escaped characters</li>
</ul>
<h3 dir="auto">3. Grouping</h3>
<ul dir="auto">
<li><code class="notranslate">(...)</code> (groups)</li>
<li><code class="notranslate">|</code> (alternation in groups)</li>
</ul>
<h3 dir="auto">4. Actions and Modifiers</h3>
<ul dir="auto">
<li><code class="notranslate">-&gt;</code> operator (<code class="notranslate">-&gt; skip</code>, <code class="notranslate">-&gt; more</code>, <code class="notranslate">-&gt; popMode</code>)</li>
<li><code class="notranslate">fragment</code> keyword</li>
<li><code class="notranslate">mode</code> declarations</li>
</ul>
<h3 dir="auto">5. Fragment References</h3>
<ul dir="auto">
<li>Support for fragment rules and references</li>
</ul>
<h2 dir="auto">Example Cases That Fail</h2>
<p dir="auto">From XMLLexer.g4:</p>
<div class="highlight highlight-source-antlr notranslate position-relative overflow-auto" dir="auto" data-snippet-clipboard-copy-content="COMMENT : '&lt;!--' .*? '--&gt;' ;
DTD : '&lt;!' .*? '&gt;' -&gt; skip ;
CharRef : '&amp;#' DIGIT+ ';' ;
Name : NameStartChar NameChar* ;
fragment DIGIT: [0-9] ;"><pre class="notranslate"><span class="pl-en">COMMENT</span> : <span class="pl-s"><span class="pl-pds">'</span>&lt;!--<span class="pl-pds">'</span></span> .*? <span class="pl-s"><span class="pl-pds">'</span>--&gt;<span class="pl-pds">'</span></span> ;
<span class="pl-en">DTD</span> : <span class="pl-s"><span class="pl-pds">'</span>&lt;!<span class="pl-pds">'</span></span> .*? <span class="pl-s"><span class="pl-pds">'</span>&gt;<span class="pl-pds">'</span></span> -&gt; skip ;
<span class="pl-en">CharRef</span> : <span class="pl-s"><span class="pl-pds">'</span>&amp;#<span class="pl-pds">'</span></span> <span class="pl-c1">DIGIT</span>+ <span class="pl-s"><span class="pl-pds">'</span>;<span class="pl-pds">'</span></span> ;
<span class="pl-en">Name</span> : NameStartChar NameChar* ;
<span class="pl-en">fragment</span> <span class="pl-c1">DIGIT</span>: [0-9] ;</pre></div>
<p dir="auto">The parser fails to parse these rules because they use quantifiers (<code class="notranslate">*?</code>, <code class="notranslate">+</code>, <code class="notranslate">*</code>) and actions (<code class="notranslate">-&gt; skip</code>) that are not currently supported.</p>
<h2 dir="auto">Acceptance Criteria</h2>
<ul dir="auto">
<li>Parser recognizes ANTLR v4 quantifier operators</li>
<li>Parser handles <code class="notranslate">.</code> (any character) in rules</li>
<li>Parser recognizes groups and alternations</li>
<li>Parser recognizes <code class="notranslate">-&gt;</code> actions</li>
<li>Parser recognizes <code class="notranslate">fragment</code> keyword</li>
<li>Parser recognizes <code class="notranslate">mode</code> declarations</li>
<li>XMLLexer.g4 parses completely with all rules in the AST</li>
<li>Test coverage for new syntax features</li>
</ul>
<h2 dir="auto">Impact</h2>
<p dir="auto">This will allow the parser to handle real-world ANTLR v4 grammars from grammars-v4 repository and other modern ANTLR projects.</p></div></div><div
